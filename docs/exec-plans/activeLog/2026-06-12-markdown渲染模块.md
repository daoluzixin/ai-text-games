# 执行日志：Markdown 渲染模块（Shadow DOM 隔离）

日期：2026-06-12

---

## 背景

AI 回复的文本包含 markdown 格式（粗体、选项列表等），但前端使用 `textContent` 直接输出，导致所有格式标记以原始文本展示，阅读体验差。用户要求在隔离环境中渲染 markdown，且方便后续扩展。

---

## 实施内容

### 1. 新增独立渲染模块 `static/md-renderer.js`

- 使用 Shadow DOM 实现完全样式隔离，每个 AI 气泡有独立的 shadow root
- 依赖 marked.js（CDN）做 markdown → HTML 转换
- 暴露 `MdRenderer` 类，支持流式 `update(text)` 调用
- 文游场景专属增强：
  - 段落首行缩进 2em + 行高 1.9，阅读体验接近小说
  - 自动识别 A/B/C/D 选项并渲染为卡片（带圆形彩色标签、hover 动效）
  - 对话引号内容自动高亮（紫色）
  - 粗体文字带底部荧光笔效果
  - 字体切换为衬线体（更有文学感）

### 2. 修改 `index.html`

- 引入 marked.js CDN + `/static/md-renderer.js`
- `addMessage()` 函数：AI 消息创建 `MdRenderer` 实例挂载到气泡
- 流式更新逻辑：从 `textContent = aiMessage` 改为 `renderer.update(aiMessage)`
- AI 气泡 CSS：`white-space: normal`、加大内边距、渐变背景、更宽（88%）

### 3. 修改 `server.py`

- 新增 `/static` 路由挂载 `static/` 目录（使用 FastAPI StaticFiles）

### 4. 更新 `docs/agents/核心架构.md`

- 目录结构新增 `static/` 及 `md-renderer.js` 说明
- 静态层描述更新
- 新增「Markdown 渲染模块」章节
