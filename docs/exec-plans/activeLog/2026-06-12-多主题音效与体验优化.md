# 执行日志：多主题音效与体验优化

日期：2026-06-12

## 本次改了什么

### 1. 状态卡片自动进度条（md-frame.html）

- 新增 `autoDetectBar(label, value)` 函数：基于关键词正则（好感度/忠诚/体力/魅力/智力/饥饿/生命值等）+ 数值提取，自动判断是否渲染为进度条
- 新增 `renderBarHtml(icon, label, value, num, max)` 共享渲染函数，统一进度条 HTML 生成
- `renderStatusItem()`（严格渲染器）：当 `type!="bar"` 时调用 `autoDetectBar` 自动检测
- `renderStatusLoose()`（宽松渲染器）：同样接入 `autoDetectBar`，不再需要 docx 显式声明 `type="bar"`

### 2. 主题按钮提示动画（index.html）

- 新增 `.theme-toggle-btn.hint-pulse` CSS 动画：粉色脉冲光晕（`@keyframes themePulse`）
- 新增 `.theme-hint-tooltip` 气泡样式：固定定位、淡入淡出、三角箭头
- 新增 `dismissThemeHint()` 函数：点击后永久消失（localStorage 记录 `theme-hint-seen`）
- 新增 `initThemeHint()` IIFE：首次访问延迟 1.5s 出现，8s 后自动消失

### 3. 游戏卡片摘要修复（server.py）

- `extract_summary()` 函数增强：
  - `skip_prefixes` 新增：`指令`、`注意`、`免责`、`必看`、`玩前`、`游戏指令`、`开始游戏` 等
  - 新增 `skip_keywords` 元组：`纯属虚构`、`与现实无关`、`请勿`、`健康游玩`、`此段不复制`、`复制到备忘录` 等
  - 新增 `heading_re` 正则：跳过 `一、`、`1.`、`第X章` 等章节标题行
- 效果：摘要从"指令内容纯属虚构…"变为真正的故事描述（如"玩家扮演一位店主…"）

### 4. 音效系统（index.html，Web Audio API 合成）

全部使用 Web Audio API 程序化合成，零外部音频文件：

- **背景音乐**：triangle 波形 + 低通滤波（800Hz 截止）+ 低音区音符（C3-C4），按主题切换调性（薰衣草=C大调、哥特=A小调、赛博=电子、森林=D大调），缓慢淡入淡出像呼吸
- **按钮点击音**：sine 波 600→400Hz 下降音 + 低通滤波，80ms 极短水滴感
- **收到回复提示音**：triangle 波 C5+E5 大三度双音 + 低通滤波，柔和叮咚
- **AI 打字键盘音**：12ms 白噪声 + bandpass 滤波（600-1000Hz），极轻触感，90-150ms 间隔循环
- **控制面板**：右上角 🔊 按钮，展开面板含音效开关、BGM 开关、统一音量滑块
- 所有设置持久化到 localStorage（`sfx-enabled`、`bgm-enabled`、`sfx-volume`）
- 首次用户交互后自动启动 BGM（浏览器 autoplay 策略兼容）

### 5. 刷新保持存档状态（index.html）

- `enterGame()` 时将 `{gameId, gameName, avatarUrl, sessionId, mode}` 存入 `sessionStorage('active-game')`
- `goBack()` 时清除该状态
- 页面初始化时 `restoreActiveGame()` 检测并自动恢复：立即切换页面 DOM + 调用 `enterGame` 读档
- 使用 `sessionStorage`（非 localStorage），关闭标签页后不残留

### 6. 读取存档速度优化

**后端 server.py**：
- `GET /api/sessions/{session_id}` 优先从内存 `sessions` 字典返回，不再每次查 SQLite
- 仅内存未命中时回退数据库查询

**前端 index.html**：
- `loadSessionHistory()` 改为渐进式渲染：
  - 第一步：用 `DocumentFragment` 批量创建所有消息的纯文本版本，瞬间显示
  - 第二步：异步逐个将 assistant 气泡升级为 iframe 富文本（每帧一个，不卡顿）
  - 用户立刻能看到内容并滚动，富文本渲染在后台渐进完成

### 7. 静态文件缓存控制（server.py）

- `serve_index()` 响应头增加 `Cache-Control: no-cache, no-store, must-revalidate`
- 确保浏览器每次拿到最新版 index.html，避免缓存旧版本
