# 执行计划体系

## 目录结构

```
exec-plans/
├── active/      ← 当前正在执行的计划
├── completed/   ← 已完成验收的计划
└── activeLog/   ← 执行日志（每步关键操作记录）
```

## 计划模板

新建计划时，在 `active/` 下创建 markdown 文件，至少包含以下内容：

```markdown
# <功能名称> 执行计划

## 状态

当前状态：待执行
可选值：待执行 / 执行中 / 完成

## 功能边界

- 本次做什么
- 本次不做什么

## 对外契约变化

- API 接口变更（如有）
- 数据结构变更（如有）

## 关联依赖

- 依赖的外部服务/库
- 需要的环境配置

## 验收标准

- [ ] 标准 1
- [ ] 标准 2

## 自反馈验证方式

按 docs/agents/自反馈工作流.md 执行，具体验证步骤：
1. ...
2. ...
```

## 状态流转

```
待执行 → 执行中 → 完成
                 ↘ （验收不通过）回到执行中
```

## 执行日志规则

执行计划时，每完成一步关键操作，都必须更新 `activeLog/` 下对应的日志文件。

执行记录至少包括：

- 当前步骤做了什么
- 是否涉及代码改动
- 验证结果（API 响应 / 日志截取）
- 如果失败：原因分析和下一步计划

## 计划完成条件

计划移到 `completed/` 的前提：

1. 所有验收标准已勾选
2. 自反馈闭环已跑通（有运行态证据）
3. lint 检查 0 FAIL
