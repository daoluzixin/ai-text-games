# 工程约束与协作规范

## 不可违反的红线

1. **API_KEY 不入库**：`.env` 已在 `.gitignore` 中，任何情况下不得将密钥硬编码到代码或提交到仓库。
2. **sessions.db 不入库**：用户对话数据属于隐私数据，不得提交。
3. **games/*.docx 不入库**：游戏文件属于用户上传的运行时数据。
4. **单文件后端原则**：当前阶段 server.py 保持单文件，不拆分模块。如果未来复杂度增长需要拆分，必须先写执行计划讨论。
5. **前端零构建原则**：index.html 保持自包含，不引入 npm/webpack/vite 等构建工具。
6. **向后兼容**：API 接口变更不得破坏已有前端调用，新增字段可以，删除/改名字段必须先写计划。

## 代码风格

| 项目 | 规范 |
|------|------|
| Python 缩进 | 4 空格 |
| 字符串 | 优先双引号 |
| 类型注解 | 函数签名必须有类型注解 |
| 注释语言 | 中文 |
| 变量命名 | snake_case |
| 常量命名 | UPPER_SNAKE_CASE |
| API 路径 | 小写 kebab-case（如 `/api/new-session`） |

## 部署约束

| 约束 | 说明 |
|------|------|
| 部署方式 | systemd 服务（`ai-text-games.service`） |
| 同步方式 | rsync 推送到 `/root/ai-text-games/`（systemd 实际运行目录，非 `/opt`） |
| 重启命令 | `ssh root@39.106.187.120 'systemctl restart ai-text-games'` |
| 日志查看 | `ssh root@39.106.187.120 'journalctl -u ai-text-games -f'` |

## 文件上传约束

| 约束 | 值 |
|------|-----|
| 允许格式 | 仅 `.docx` |
| 大小限制 | 10MB |
| 存储位置 | `games/` 目录 |
| 内容校验 | 必须能被 python-docx 解析且包含文字内容 |

## Git 提交规范

- commit message 使用中文
- 格式：`<类型>: <简述>`
- 类型：feat / fix / refactor / docs / deploy / chore
- 示例：`feat: 支持删除游戏功能`

## 文档维护规则

- 改了 API 接口 → 更新 `docs/agents/核心架构.md` 的接口清单
- 改了部署配置 → 更新 `docs/agents/目元信息.md` 的部署信息
- 新增功能 → 先在 `docs/exec-plans/active/` 写执行计划
- 完成功能 → 将计划移到 `docs/exec-plans/completed/`
