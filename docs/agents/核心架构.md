# 核心架构

## 技术栈

| 层 | 技术 |
|----|------|
| 后端 | Python 3.10+ / FastAPI / Uvicorn |
| 前端 | 单文件 `index.html`（原生 HTML/CSS/JS，无构建工具） |
| 数据库 | SQLite（会话持久化） |
| LLM | DeepSeek（OpenAI 兼容接口，可替换为任何兼容 API） |
| 部署 | systemd 直接运行 / Docker Compose 可选 |

## 目录结构与模块职责

```
ai-text-games/
├── server.py          ← 后端唯一入口，FastAPI 应用
├── index.html         ← 前端单页应用
├── requirements.txt   ← Python 依赖声明
├── .env               ← 运行时配置（API_KEY 等，不入库）
├── .env.example       ← 配置模板
├── Dockerfile         ← Docker 镜像定义
├── docker-compose.yml ← Docker Compose 编排
├── deploy.sh          ← 一键部署脚本
├── games/             ← 游戏 .docx 文件目录（运行时数据，不入库）
├── avatars/           ← 游戏头像 SVG 文件
├── static/            ← 前端独立模块（JS/CSS，由后端 /static 路由提供）
│   └── md-renderer.js ← Markdown 渲染引擎（Shadow DOM 隔离）
├── sessions.db        ← SQLite 数据库（运行时生成，不入库）
├── AGENTS.md          ← Agent 导引文件
├── docs/              ← 知识文档与执行计划
│   ├── agents/        ← 长期知识文档
│   └── exec-plans/    ← 执行计划体系
├── scripts/           ← 自动化脚本
└── .catpaw/rules/     ← Agent 规则自动注入
```

## 后端架构（server.py）

server.py 是一个单文件 FastAPI 应用，职责分层如下：

| 层 | 职责 |
|----|------|
| 配置层 | 从 `.env` 读取 API_KEY、MODEL_NAME、PORT 等 |
| 数据层 | SQLite 持久化会话历史（init_db / save_session / load_sessions_from_db） |
| 业务层 | 游戏加载（load_games）、头像分配（assign_avatar）、对话管理 |
| API 层 | RESTful 接口（/api/games, /api/chat, /api/sessions 等） |
| 静态层 | 返回 index.html + 挂载 /static 目录（md-renderer.js 等前端模块） |

## API 接口清单

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/api/games` | 获取游戏列表 |
| GET | `/api/avatars/{filename}` | 获取头像文件 |
| GET | `/api/sessions` | 列出所有存档 |
| GET | `/api/sessions/{session_id}` | 获取存档详情 |
| GET | `/api/new-session` | 生成新 session_id |
| POST | `/api/chat` | 发送消息，流式返回 AI 回复 |
| POST | `/api/reset` | 重置会话历史 |
| POST | `/api/upload` | 上传 .docx 游戏文件 |
| DELETE | `/api/games/{game_id}` | 删除游戏 |
| DELETE | `/api/sessions/{session_id}` | 删除存档 |

## 数据流

```
用户输入 → POST /api/chat → 构建 messages（system prompt + 历史）
→ AsyncOpenAI 流式调用 → StreamingResponse 逐 chunk 返回前端
→ 流结束后 save_session 落盘 SQLite
```

## 前端架构

index.html 是单页应用，通过 fetch 调用后端 API，使用 ReadableStream 处理流式响应。

### Markdown 渲染模块（static/md-renderer.js）

AI 回复的 markdown 文本通过独立的渲染模块在 Shadow DOM 中隔离渲染，确保样式不互相污染。

| 特性 | 说明 |
|------|------|
| 隔离方式 | Shadow DOM（每个 AI 气泡独立 shadow root） |
| 渲染引擎 | marked.js（CDN 引入） |
| 流式支持 | `MdRenderer.update(text)` 方法，流式追加时反复调用 |
| 文游增强 | 自动识别 A/B/C/D 选项并卡片化、对话引号高亮、段落首行缩进 |
| 降级策略 | marked.js 加载失败时退回纯文本 |
| 扩展方式 | 修改 `static/md-renderer.js` 即可，不需动 index.html 主逻辑 |
